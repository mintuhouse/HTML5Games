Why is design important?

  • People take you more seriously

    Especially important if you want to get the word out, or want journalists to write about you

  • Better usability

    Even if things work great on the back end, users need a good way to interact with the application

  • Consistency across browsers

What is HTML?

Hyper Text Markup Language is used to create and identify content on the page

In other words, the language of "sandwiching content with identifying tags"

<html> <head> <meta charset="utf-8"> <link rel="stylesheet"> <script type="text/javascript"> </head> <body> <div> Stuff </div> </body> </html>

Laying out page elements

Generally two ways to position elements:

  1. Absolute positioning

    Not recommended, but useful for certain situations

  2. Relative positioning

    Allows you to float elements, use more of a fluid layout

Ways to use the display property:

  • display: none

    Hide elements

  • display: block

    Default value, element takes up whole width

  • display: inline

    Element spans only its contents

Grid systems

Provide a structure for all elements on a page to follow

  • Most common: 960 grid system

Contrast in Design

  • General rule: go big or go home.

  • With Color

    Use strong contrast between text color and background color

    If in doubt, see what your design looks like in grayscale

  • With Typography

    When mixing fonts, change up the font style + weight

Advanced CSS

Let's start with sprites

  • The problem:

    We don't want to wait for all the images on the page to load

  • The solution:

    Put them all in one file

  • Example

    <div class="ui" id="up"></div> <div class="ui" id="down"></div> <div class="ui" id="left"></div> <div class="ui" id="right"></div> 
    .ui {
	width: 40px;
	height: 40px;
	float: left;
	margin: 5px;
}

#up { background-position: 0px 0px; }

#down { background-position: 0px -40px; }

#left { background-position: 0px -80px; }

#right { background-position: 0px -120px; }

Cross-Browser Support

  • Why you should stop designing for IE

  • Often times, you'll have to include multiple CSS rules that do the same thing

    .button {
	box-shadow: 0px 1px 8px #999;
	-moz-box-shadow: 0px 1px 8px #999;
	-webkit-box-shadow: 0px 1px 8px #999;
	-khtml-box-shadow: 0px 1px 8px #999;

	border-radius: 5px;
	-moz-border-radius: 5px;
	-webkit-border-radius: 5px;
	-khtml-border-radius: 5px;
}
